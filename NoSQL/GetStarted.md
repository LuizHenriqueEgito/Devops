# NoSQL (not only SQL)
Neste banco de dados, os dados não são armazenados em formato de tabela, geralmente são armazenados em formato de documentos (chave e valor). Este tipo de banco de dados veio para ser uma solução para se trabalhar com dados **não estruturados**, como `JSON` por exemplo.
É uma alternativa (e não um substituto) aos bancos de dados relacionais.
- MongoDB
- Cassandra
- Neo4j
- Redis

## Quando escolher SQL ou NoSQL
O ideal é utilizar SQL quando: *relacionamento* e *consistência* são críticos.

Já NoSQL deve ser escolhido quando *escala* e *flexibilidade* são importantes.
